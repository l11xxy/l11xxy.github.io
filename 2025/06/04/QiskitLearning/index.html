现在为电路添加上量子门

可以看到我们在量子线路上添加了两个量子门，一个是H门，一个是CX门。

H门只需要一个量子比特输入，所以只出现在q0这条线路上，CNOT门是一个用于创建纠缠的量子比特的门，C指Controlled，受控的，所以有两个量子比特输入，第一个量子比特作为控制比特，第二个作为被控制的比特。

测量方式

我们现在有了初始的量子比特，也对量子比特进行了操作，那么我们最终是需要对量子比特进行测量的，下面给出qiskit中几种进行测量的方式。

第一种测量方式，使用AerSimulator

首先给出一个概念，量子比特在测量前可能处于叠加或者纠缠态，比如|ψ⟩ = α|0⟩ + β|1⟩，但是我们无法直接看到这个态，当我们使用Z基进行测量的时候，这个处于叠加态的量子态会塌缩为一个经典结果，0或者1。

这个过程是不可逆的，并且每次测量都可能得到不同的结果。

所以在qiskit的设计中中必须使用一个经典比特来储存测量结果，所以我们必须对前面的代码进行一点改变，在QuantumCircuit中添加经典比特才能进行测量。

然后我们需要导入qiskit_aer来对电路进行模拟，qiskit_aer是用于在经典计算机上高效地模拟量子电路的执行和噪声的后端库。

自 Qiskit 1.0 起，Aer被从主库中分离出来，成为一个独立安装的包：qiskit_aer。所以在有些教程中让在Qiskit中导入，实际上已经过时挺久的了😭。

qiskit_aer提供了多个模拟器后段，我只看了第一个，所以后边我们就用第一个来演示。

模拟器名称 功能说明
AerSimulator 通用模拟器，支持噪声、自定义运行配置等（Qiskit 1.0 之后推荐使用）
StatevectorSimulator 仅用于无测量、无噪声的态向量模拟（适用于理论分析）
UnitarySimulator 输出电路的总酉矩阵（主要用于调试）

  1. 在之前的qiskit中两个量子线路可以通过+进行组合，但是在现在较新的版本下已经变为了使用以下几个方法进行组合

    • QuantumCircuit.append()

      Append one or more instructions to the end of the circuit, modifying the circuit in place.

      The qargs and cargs will be expanded and broadcast according to the rules of the given Instruction, and any non-Bit specifiers (such as integer indices) will be resolved into the relevant instances.

      If a CircuitInstruction is given, it will be unwrapped, verified in the context of this circuit, and a new object will be appended to the circuit. In this case, you may not pass qargs or cargs separately.

    • QuantumCircuit.compose()

      GitHub

      Apply the instructions from one circuit onto specified qubits and/or clbits on another.

      Note

      By default, this creates a new circuit object, leaving self untouched. For most uses of this function, it is far more efficient to set inplace=True and modify the base circuit in-place.
