这里我们可以看到添加了三个函数，但是他们的参数都不同

fn new() -> Self为结构体关联函数，它不依赖已经创建的实例，可以直接调用。

fn hardmard(&mut self)的参数是 &mut self，属于实例方法，必须通过已经创建好的 qubit 实例调用。在调用时 Rust 会隐式地把当前这个 qubit 对象传进来，self 就代表此刻调用该方法的这个量子比特对象。

但是此方法参数是&mut self ，而fn measure的参数是&self这又是为什么呢？

因为方法fn hardmard(&mut self)内部执行了 self.superposed = true，要修改结构体内部的字段，而根据Rust 语法规则，想要修改变量内部数据，必须使用可变引用 &mut。如果只是读取数据不修改（比如 measure(&self)），只用不可变引用 &self 就可以。

同时还有一个值得注意的地方，measure函数最后一行表达式没有加分号、没有写 return，这是rust语法的一个特性：函数末尾表达式会隐式作为返回值。

Rust 不是传统面向对象编程语言，没有类、没有继承，但是通过Rust 结构体 + impl可以模拟实现类的封装

Rust 所有权

rust没有了面向对象，那么有什么和其他编程语言不一样的东西吗，有的，rust（ownership）使用了所有权的概念。

C/C++ 需要通过malloc和free来对内存进行管理，Java通过garbage collector自动进行内存回收，rust通过所有权系统管理内存。

所有权有以下三条规则：

  • Rust 中的每个值都有一个变量，称为其所有者。
  • 一次只能有一个所有者。
  • 当所有者不在程序运行范围时，该值将被删除。

这三条规则是所有权概念的基础。

Move（移动）

仍然使用刚才的例子做演示，Rust 的所有权机制使得在默认情况下，赋值操作会移动所有权，而不是复制。

这和 C++ 的浅拷贝看起来很像，但关键区别是：移动后原变量彻底作废，编译器会在编译期阻止你继续使用它。量子比特的状态被”传递”了，而不是被偷偷复制了一份。

借用（Borrowing）

大多数时候你不需要拿走所有权，只想读一下或者改一下。这就是前面反复出现的 &self 和 &mut self 的本质。

借用有两条规定，编译器会严格执行：

  1. 同一时刻，要么有任意多个不可变借用 &T，要么只有唯一一个可变借用 &mut T，二者互斥。
  2. 所有者被释放后借用也会无效，编译器会自动检查这一点。

编译器会在编译时完成对这两条规则的检查，从而降低运行时开销。

Trait（特性）

前面我们用 struct + impl 封装了单个 qubit 的行为。但量子计算里不止 H 门，还有X门、Z门、CNOT门等。我们希望所有门都遵循同一个定义框架，Rust 里这个框架叫 Trait。

Rust 同一个类可以实现多个特性，每个 impl 块只能实现一个。
